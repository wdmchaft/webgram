Class Webgram.ControlPoint

The abstract base class for all control points. A control point is a zoom-independent element on the canvas that allows the user to easily interact with a drawing element. Each control point instance belongs to a drawing element.
Defined in: control-point.js.

{Webgram.DrawingElement} drawingElement

This is a reference to the element that the control point belongs to. Its value is null until the control point is added to the drawing element using Webgram.DrawingElement#addControlPoint.

{Webgram.Event} onAdd

An event that is triggered when the control point is added to a drawing element, using Webgram.DrawingElement#addControlPoint.
Handlers receive the following arguments: (drawingElement).

{Webgram.Event} onBeginMove

An event that is triggered when the control point moving begins.
Handlers receive no arguments.

{Webgram.Event} onChangeFocus

An event that is triggered when the control points's focus type is changed, by a call to Webgram.ControlPoint#setFocusType.
Handlers receive no arguments.

{Webgram.Event} onEndMove

An event that is triggered when the control point moving ends.
Handlers receive no arguments.

{Webgram.Event} onMouseEnter

An event that is triggered whenever the mouse enters the area of this control point. Webgram.ControlPoint#pointInside method is used to evaluate this condition.
This is the equivalent of onMouseOver JavaScript event.
Handlers receive no arguments.

{Webgram.Event} onMouseLeave

An event that is triggered whenever the mouse leaves the area of this control point. Webgram.ControlPoint#pointInside method is used to evaluate this condition.
This is the equivalent of onMouseOut JavaScript event.
Handlers receive no arguments.

{Webgram.Event} onMove

An event that is triggered whenever the control point is moved.
Handlers receive the following arguments: (point)

{Webgram.Event} onRemove

An event that is triggered when the control point is removed from a drawing element, using Webgram.DrawingElement.remControlPoint.
Handlers receive the following arguments: (drawingElement).

{Webgram.Event} onSelect

An event that is triggered when the control point is selected by the active Webgram.DrawingControl.
Handlers receive no arguments.

{Webgram.Event} onUnselect

An event that is triggered when the control point is unselected by the active Webgram.DrawingControl.
Handlers receive no arguments.

{Webgram.Event} onUpdate

An event that is triggered when the control point is updated.
Handlers receive the following arguments: (deltaX, deltaY)

{Number} radius

A control point has a radius, used to determine if the mouse is inside inside it. This radius does not change with the zoom level.

Method Detail
computeAnchor()

This method is responsible for the actual computing of the anchor. Implement this method in concrete control point classes and make it return the anchor point, relative to the parent of this control point's element.

(must be overridden)

draw()

This method is responsible for the way the control point looks on the canvas. Concrete drawing elements should implement this method by calling the various available drawing primitives (draw* methods), followed by a call to paint.
Figures drawn by a control point are not affected by the zoom.

(should be overridden)

{Webgram.Geometry.Point} getAnchor()

Returns the anchor of this control point. The anchor represents the point where the control point is pinned onto its drawing element. The coordinates of this anchor are relative to the element's parent. In case the anchor is null (read: invalid), this method calls Webgram.ControlPoint#update which in turn makes a call to Webgram.ControlPoint#computeAnchor to obtain the value of the anchor.

{Webgram.Geometry.Point} the value of the anchor

paint(strokeStyle, fillStyle)

This method must be called after using one or more path-related drawing primitives. It actually commits the path-drawing operations that were previously queued. No path operations should remain "unpainted" (i.e. with no call to paint after them).
If strokeStyle is specified, a stroke is drawn along the path, using the given style. If strokeStyle is undefined, the control points's stroke style is used (Webgram.ControlPoint#getStrokeStyle). If strokeStyle is null, no stroke is drawn along the path at all. The same algorithm is used for filling the path, using the fillStyle argument.

Parameters:
{Webgram.Styles.StrokeStyle} strokeStyle
the stroke style to use when painting the primitives
{Webgram.Styles.FillStyle} fillStyle
the fill style to use when painting the primitives
