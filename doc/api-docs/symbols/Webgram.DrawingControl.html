Class Webgram.DrawingControl

This is the base class for all drawing controls. A drawing control handles the mouse and keyboard events received by the webgram canvas. A webgram can have many associated drawing controls but only one of them can be active at once.
Defined in: drawing-control.js.

Method Detail
activate()

Activates the drawing control. If this drawing control is already active, this method does nothing. If there's another active drawing control, it will be deactivated first.

(should not be overridden)

{Webgram.DrawingControl} activatePrevious()

Each drawing control remembers the drawing control that was previously active, before it became active. This method reactivates the previous drawing control, if any.

(should not be overridden)

Returns:
{Webgram.DrawingControl} the drawing control that was reactivated
deactivate()

Deactivates the drawing control. If this drawing control is not active, this method does nothing.

(should not be overridden)
