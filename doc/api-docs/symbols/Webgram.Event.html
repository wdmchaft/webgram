Class Webgram.Event

Manages the connection (binding) between a library event and one or more handlers. Events can be of any origin and kind, not only those that come from the mouse or the keyboard.
Defined in: event.js.

Method Detail
bind(handler)

Adds a handler to the event. This function doesn't check for duplicates, so if a handler is bound more times to the same event, it will be called more times when the event is triggered.
The handler functions are called with this set to the owner object of the event (Webgram.Event#object).
Any additional arguments passed to this method will be passed to the handler function when the event is triggered.

Parameters:
{Function} handler
the function to bind as handler to the event
{String} toString()

Returns a string representation of this event.

Returns:
{String} the string representation of this event
{any} trigger()

Triggers the event. All the handler functions will be called in the order of binding, until one of them returns false.

{any} the last non-false value returned by the handlers
