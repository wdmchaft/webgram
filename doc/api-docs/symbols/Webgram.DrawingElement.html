Defined in: drawing-element.js.

{Number} minPointCount

This field specifies the minimum number of shape points that this drawing element needs in order to be consistent, i.e. to function properly. You should only set this field when constructing the object. Set it to a value greater than 0.

{Webgram.Event} onAdd

An event that is triggered when the element is added to a Webgram.DrawingElements.ContainerElement.
Handlers receive the following arguments: (containerElement).

{Webgram.Event} onBeginRotate

An event that is triggered when the user begins the rotation of this element. This event only occurs when the element is rotated using Webgram.ControlPoints.RotateControlPoint.
Handlers receive no arguments.

{Webgram.Event} onBeginShapeChange

An event that is triggered when the shape starts to change.
Handlers receive no arguments.

{Webgram.Event} onBeginStyleChange

An event that is triggered when the user begins changing one of the styles.
Handlers receive no arguments.

{Webgram.Event} onChange

An event that is triggered after the style, the shape or the rotation angle of this drawing element is modified. In addition to these, any additional attributes that contribute to the json form of the element should trigger this event. This event is linked with the more general Webgram#onDrawingElementChange event
Handlers receive no arguments.

{Webgram.Event} onChangeFocus

An event that is triggered when the element's focus type is changed, by a call to Webgram.DrawingElement#setFocusType.
Handlers receive no arguments.

{Webgram.Event} onCopy

An event that is triggered when the element is copied (to clipboard).
Handlers receive no arguments.

{Webgram.Event} onCreate

An event that is triggered after the element is created by the Webgram.DrawingControls.CreateDrawingControl.
Handlers receive no arguments.

{Webgram.Event} onDuplicate

An event that is triggered when the element is created using duplicate action.
Handlers receive no arguments.

Handlers receive no arguments.

{Webgram.Event} onEndShapeChange

An event that is triggered when the shape finishes to change.
Handlers receive no arguments.

{Webgram.Event} onEndStyleChange

An event that is triggered when the user finishes changing one of the styles.
Handlers receive no arguments.

Handlers receive no arguments.

An event that is triggered when the element is moved, either by the user, or by explicitly calling Webgram.DrawingElement#moveTo. This event is not triggered when the shape is changed, since such a change is considered a change in shape rather than a move.
