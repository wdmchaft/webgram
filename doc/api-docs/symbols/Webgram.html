Class Webgram

The root class of this library.
Defined in: webgram.js.

Class Detail
Webgram(htmlElement, canvas, settings)
Parameters:
{HtmlElement} htmlElement
the HTML element to be used to receive the mouse and keyboard events; this is normally the HTML Canvas element whose 2D context is used to create the canvas object passed as the second argument
{Webgram.Canvas} canvas
the canvas to be used by this instance of webgram to paint everything.
{Object} settings
an object with settings that will replace the default ones
Field Detail
{Webgram.Canvas} canvas

The canvas used to paint everything. This is basically a wrapper around the HTML Canvas element.

{Webgram.DrawingControls.CreateDrawingControl} createDrawingControl

A webgram instance automatically gets a creation drawing control.

{HTMLElement} htmlElement

The HTML element used to receive the JavaScript keyboard and mouse events. This is usually the HTML Canvas element whose 2D context was used to create the canvas.

<static>
{String} Webgram.imgPath

The root path (url) to the webgram's images. The value of this field defaults to the 'images/' string appended to Webgram.jsPath. This could be changed if the webgram images are served from a different path than the JavaScript files.

<static>
{String} Webgram.jsPath

The path (url) to the main webgram JavaScript file. The value of this field is determined automatically and should not be altered.

{Webgram.Event} onDrawingElementAdd

An event that is triggered when a drawing element is added to the webgram, either directly using Webgram#addDrawingElement or indirectly, by calling the Webgram.DrawingElements.ContainerElement#addDrawingElement on a container element already added to the webgram. Handlers receive the following arguments: (drawingElement).

{Webgram.Event} onDrawingElementCreate

Handlers receive the following arguments: (drawingElement).

{Webgram.Event} onDrawingElementIndexChange

An event that is triggered when a drawing element's position inside its parent changes, either by calling Webgram#setDrawingElementIndex/Webgram.DrawingElements.ContainerElement#setDrawingElementIndex or by adding/removing other drawing elements to/from a container element added to the webgram. Handlers receive the following arguments: (drawingElement, index).

{Webgram.Event} onDrawingElementInteract

An event that is triggered when a Webgram.Connectors.Connector is connected/disconnected to/from a drawing element.
Handlers receive the following arguments: (connector, drawingElement, type), where type is either 'connect' or 'disconnect'.

{Webgram.Event} onDrawingElementRemove

An event that is triggered when a drawing element is removed from the webgram, either directly using Webgram#remDrawingElement or indirectly, by removing a drawing element with Webgram.DrawingElements.ContainerElement#remDrawingElement from a container element added to the webgram. Handlers receive the following arguments: (drawingElement).

{Object} settings

The webgram library holds a reference to an object with the so-called global settings, manipulable using the Webgram#getSetting and Webgram#setSetting methods. These settings are accessible in all drawing elements, control points and drawing controls, using their corresponding methods with the same names as the ones described above.

addDrawingElement(drawingElement, where)

Adds a drawing element to the root container of the webgram.

(should not be overridden)

Parameters:
{Webgram.DrawingElement} drawingElement
the drawing element to add
{any} where
indicates where to add the element:
  • if a Webgram.DrawingElement is given, the element is added after that element
  • if a number is given, the element is added at that position
  • if null or undefined is given, the element is added at the end
attachHandlers()

Registers the various handler functions for the JavaScript events. You must explicitly call this method after creating a webgram instance if you want it to be interactive (i.e. to respond to user mouse/keyboard events).

(should not be overridden)
